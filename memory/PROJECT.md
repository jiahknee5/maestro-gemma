## [PM] Spec — 2026-04-04

### 1. Problem Statement

Arthur practices violin 30 minutes a day without a teacher present. Private violin lessons cost $80–150/hour — families that can't afford daily instruction leave their child practicing alone, with no one to catch the bad habits that become permanent: raised bow shoulder, collapsed left wrist, intonation drift. By the next lesson, the habit is ingrained and must be unlearned. This is not a niche problem: 30 million children study an instrument in the US and most quit within two years, in part because unsupervised practice is frustrating and unproductive. Maestro Gemma puts a Gemma 4 AI coach in the room for every practice session — watching posture via Apple Vision, hearing pitch via AVFoundation, and providing natural language technique feedback in real time using Gemma 4 E2B on-device, with deeper reasoning routed to Gemma 4 26B on a local server. The cost of not solving this: kids develop bad habits, get discouraged, and quit.

### 2. Success Criteria

**Practice Screen — Coaching Overlay**
- Happy: Given camera + mic permissions granted and Cactus model loaded, when a Practice session starts, then the camera feed is live within 2s, the Apple Vision posture skeleton overlay is visible within 2s, and the first Gemma 4 coaching message appears within 3s of session start.
- Happy: Given a student playing with a raised bow shoulder visible in frame, when the frame is analyzed by Gemma 4 E2B, then the coaching overlay updates with a posture correction in child-appropriate language (e.g. "Try dropping your right shoulder a little") within 1s of the frame being processed.
- Failure: Given camera permission denied, when the Practice screen opens, then iOS permission prompt is shown; if permanently denied, a settings-redirect message is shown. No crash.
- Failure: Given Cactus model fails to load, when a session is started, then a non-blocking warning banner appears ("AI coaching unavailable"), pitch and posture overlay still function, and the session records normally.

**Practice Screen — Pitch Display**
- Happy: Given a student playing an A4 (440Hz), when audio is analyzed via AVFoundation, then the pitch display shows "A4" with cents deviation (e.g. "+5¢") updating in real time at ≥10Hz.
- Failure: Given silence or ambient noise only, when no pitch is detected, then the pitch display shows "—" (no note), not a false reading.

**Ask Coach**
- Happy: Given Studio reachable (GET /api/tags responds in <1s), when user submits "Why does my bow sound scratchy?", then Gemma 4 26B response is displayed in the modal within 3s.
- Happy: Given Studio unreachable, when Ask Coach is submitted, then Gemma 4 E2B on-device response is shown with an "Offline mode" indicator badge.
- Failure: Given empty text field, when submit is tapped, then submit button is disabled; no request is fired.
- Failure: Given Ollama response takes >10s, when timeout occurs, then "Taking longer than expected — try again" message shown with retry button.

**Session Summary**
- Happy: Given a session of ≥60 seconds with ≥3 FeedbackEvents logged, when session is stopped, then SessionSummaryView shows score rings, an AI-generated paragraph summary, and top 3 issues by frequency within 5s.
- Happy: Given Studio reachable, when "Generate Practice Plan" is tapped, then a 5-day PracticePlan is generated by Gemma 4 26B and PracticePlanView opens.
- Failure: Given session duration <30s, when session is stopped, then "Session too short for a summary — keep practicing!" is shown instead of summary.
- Failure: Given Studio unreachable at session end, then summary is generated on-device (shorter) with an "Offline — limited summary" indicator.

**Practice Plan**
- Happy: Given ≥1 completed session in SwiftData, when PracticePlanView is opened, then a 5-day plan with daily exercise descriptions is shown.
- Failure: Given zero sessions in history, when Practice Plan is accessed, then empty state: "Complete a practice session to generate your first plan."
- Failure: Given Studio unreachable when plan is requested, then deferred state: "Practice plan available when connected to home network."

**Teacher Report**
- Happy: Given ≥1 session this week, when "Share with Teacher" is tapped, then a structured report (issues by frequency, progress trend, recommended focus) is generated by Gemma 4 26B and the iOS share sheet opens with the report as text or PDF.
- Failure: Given Studio unreachable, when Teacher Report is requested, then: "Teacher report requires home network connection." No fallback — 26B quality is required.
- Failure: Given zero sessions this week, when Teacher Report is opened, then empty state: "No practice sessions this week."

**History**
- Happy: Given 5 completed sessions across 3 different days, when History tab is opened, then calendar shows indicator dots on those dates; tapping a date shows the session list for that day.
- Failure: Given zero sessions, when History tab is opened, then empty state: "Your practice history will appear here. Start your first session!"

### 4. Edge Cases

**Practice Screen**
- Camera permission permanently denied → settings redirect message, no crash
- Cactus model not loaded (slow device, first launch) → session still records; coaching overlay shows "Loading AI coach..." spinner
- Very fast playing or open strings → pitch display may show brief noise between notes; acceptable, no special handling needed
- iPhone gets hot during long session → iOS thermal throttling may slow inference; show no error, degraded gracefully

**Ask Coach**
- User submits multiple questions in rapid succession → queue requests, process one at a time; disable submit while processing
- Response truncated mid-sentence by max_tokens → show full response as-is; 500 token limit in Ollama options is sufficient for coaching answers
- Special characters in question (emoji, non-ASCII) → pass through as-is; Gemma 4 handles Unicode

**Session Summary**
- Session has 0 FeedbackEvents (Cactus never loaded) → show "Session recorded — AI coaching was unavailable" state; show duration only, no score rings
- 26B summary times out → show raw issue log without narrative summary; "AI summary unavailable — here are your logged issues"
- User taps "Generate Practice Plan" twice rapidly → debounce, fire only once

**Practice Plan**
- Only 1 session in history → generate plan but note "More sessions will improve recommendations"
- Session history is very old (>30 days) → still generate, but don't mention stale data to child; coach handles context

**Teacher Report**
- PDF generation fails → fallback to plain text via share sheet; never show a blank share sheet
- Share sheet is cancelled by user → no error, return to summary silently

**History**
- 100+ sessions → SwiftData query must fetch paginated; don't load all sessions into memory
- Duplicate sessions (clock change edge case) → show both; don't deduplicate silently
